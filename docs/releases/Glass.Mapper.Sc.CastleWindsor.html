Changes for the Glass.Mapper.Sc.CastleWindsor project:

3.0.1
First Release
3.0.3
  • Configuration changes to support inferred types by template ID
3.0.3.8 - Hotfix
  • Fixed lazy loading loop issue when using Castle Windsor construction. If you are using Castle Windsor to create object (see Tutorial 13 please upgrade to this release.
3.1.0.0
  • Updated to support HtmlEncodedString
3.1.1.0
  • Refactored attribute configuration. Glass.Mapper.Sc.Configuration.Attributes.SitecoreAttributeConfigurationLoader is now obsolete. Property attributes are now supported without the need for the class attribute.
3.1.2
  • Added support for SitecoreItem attribute, this allows the item being mapped to itself mapped to a property.
  • Added support for the ChildrenCast property type. This allows child items to be cast to different target types.
3.1.3
  • Now compiles .Net 4 and .Net 4.5 assemblies.
3.1.4
  • Fixed bug where the Page Editor functionality when dealing with multi-lingual sites was different to Sitecore
3.1.5
  • Stopped race condition caused by a type being registered twice with the Windsor Container.
3.2.0
  • Updated handlers to use the custom No Tracking lifestyle in Castle Windsor.
  • Update the Windsor objection creation pipeline task to use the custom No Tracking lifestyle.
