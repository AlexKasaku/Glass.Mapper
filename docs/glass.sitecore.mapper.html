For those who have used Glass.Sitecore.Mapper the biggest question is going to be what has changed, this page attempts to detail all the new features:

Abstractions
To make Glass.Mapper a platform that could be used across multiple CMS's we have had to abstract a lot of the logic. This means that when you create for example a data mapper you will want to cast the configuration or context to the CMS specific version.
Multiple Assemblies
There are now multiple assemblies for Glass.Mapper. The Glass.Mapper assembly contains the generic mapping code and the Glass.Mapper.Sc contains the Sitecore specific implementations. You will need to include both in your project.
SitecoreClass -> SitecoreType
The type SitecoreClass as now been renamed SitecoreType, we made this change because you can map more than just classes with Glass.Mapper.
More Unit Tests
We have added more unit tests and organised them in a more controlled and logic manner.
