Building

I have added the binary file to the "Depends" directory but they still show as an error in Visual Studio
Make sure you have downloaded the correct version of Sitecore (see Glass Dependencies for the current version in use) and that you have copied both the "bin" and "bin_Net4" directories form the website root. These directories need to exist in the "Depends" directory for Visual Studio to reference them correctly.
My build is failing with an error like [Error 200 Unable to copy file "xxxx" to "xxxx". Could not find a part of the path 'xxx'. C:\Program Files (x86)\MSBuild\HedgehogDevelopment\SitecoreProject\v9.0\HedgehogDevelopment.SitecoreProject.targets 189 6 xxxx]

This is due to TDS not being configured for your sites location on disk. Go to the Solution explorer and edit the TDS projects

Click the Build option and edit the "Sitecore Deploy Folder". This needs to point to the site of your disk.

I like to have my own build configuration so you don't need to touch the default one.
