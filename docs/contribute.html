We welcome contributions from anyone in the community to help us make Glass.Mapper bigger and better. If you have a feature, fix or want to make Glass.Mapper work with a completely new CMS fork the solution on Github and send us a pull request. The information below should help you get started with the solution:

Glass.Mapper follows the branching strategy suggested by Vincent Driessen on his blog. Please read this before contributing to Glass.Mapper.

To make like easier Vincent wrote an add on to Git called Git flow. It is recommended that you download and install this extension to make it easier to contribute.

To make it easier to contribute we have an solution for each CMS or a single overall solutions, see Solutions & Projects.
