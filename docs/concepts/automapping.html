Glass.Mapper supports the ability to auto-map properties from your classes. When auto-mapping you allow Glass.Mapper to decided what properties are fields, meta data or links to other items.

To activate auto-mapping for attributes you just need to set the AutoMap property to true on SitecoreType attribute: 

    [SitecoreType(AutoMap =  true)]
    public class HomePage
    {
        public virtual string Title { get; set; }
        public virtual string MainBody { get; set; }
    }

Or if you are using fluent configuration: 

    var homePage = loader.Add<HomePage>().AutoMap();

Auto-mapping does put some limitations on you field names because your field name need to be valid in C#, however if for some reason your field name can't be valid in C# you can add a custom mapping for the field as detailed further down.

By default the auto-mapping will treat a property as a field unless it has a particular name detailed in the table below:

Sitecore Umbraco
Id Maps the items Id.
Parent Maps the items parent.
Children Maps the items children.

The auto-mapper will automatically map the properties that match the enum values on the following types:

  • Glass.Mapper.Sc.Configuration.SitecoreInfoType

For example the following class maps the items Url, Name and Version: 

    [SitecoreType(AutoMap =  true)]
    public class HomePage
    {
        public virtual string Url { get; set; }
        public virtual string Name { get; set; }
        public virtual int Version { get; set; }
    }

Auto-mapping allows you to mix statically mapped and properties with the auto-mapped properties, see the example below: 

    [SitecoreType(AutoMap =  true)]
    public class HomePage
    {
        public virtual string Title { get; set; }
        public virtual string MainBody { get; set; }
        
        [SitecoreQuery("/sitecore/content/home/news/*")]
        public virtual IEnumerable<News> { get; set; }
    
        [SitecoreField("Featured Image")]
        public virtual Image FeaturedImage { get; set; }
    }

Auto-mapping is a great way to get a model quickly created with the minimal amount of code.
