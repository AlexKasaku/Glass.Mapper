When inserting images into the page you will want to control the size of the image and use Sitecore's ability to automatically resize images, this can all be handled through Glass.

For this tutorial I will assume I have the following model which we used in an earlier tutorial: 

    [SitecoreType(AutoMap = true)]
    public class NewsArticle : ContentBase
    {
        [SitecoreField]
        public virtual string Abstract { get; set; }

        [SitecoreField]
        public virtual string MainBody { get; set; }

        [SitecoreField]
        public virtual Image FeaturedImage { get; set; }

        [SitecoreField]
        public virtual DateTime Date { get; set; }
    }

Rendering an image is very straight forward using the GlassHtml helper methods: 

    <%=GlassHtml.RenderImage(Model.FeaturedImage) %>

When rendered this will output the following HTML: 

    <img src="/~/media/Site/catonsofa.ashx" alt="Cat on sofa">

Sometimes we want to control the size of the image, we can do this by passing in a second set of parameters in the form of a name value collection: 

    <%=GlassHtml.RenderImage(Model.FeaturedImage, new NameValueCollection{{"width", "80"},{"height","50"}}) %>

This will output the following HTML, notice the additional parameters added to the image URL: 

    <img src="/~/media/Site/catonsofa.ashx?w=80&h=50" width="80" height="50" alt="Cat on sofa">

You can get a complete list of the parameters support from Brian Pedersen blog.

The final step is to make the image work with the page editor using the Editable method: 

    <%= Editable(news, x=>x.FeaturedImage, x=> GlassHtml.RenderImage(x.FeaturedImage, new NameValueCollection{{"width", "80"},{"height","50"}})) %>

The editable method has an a 3 parameter overload, the first takes the target model, the second the is a lambda indicating which property you want to make editable and the third is how the content should be rendered when the page isn't in edit mode.
